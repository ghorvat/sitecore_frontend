00_base folder

All base styling should go in this folder that includes custom variables, mixins and general typography. Everything that seems general and is not specific to us it should go into this folder.

01_utility folder

All utility styles you might need during the duration of the project should go in this folder. Some of the utility classes you could put is for no paddings, no margins and so on there is an example given

02_molecule folder

All standalone element styles should go into this folder. That means if you have general button style for example All buttons have a modifier of hover which is red color and there are three type of button length this would all go in here. One class and scss file would be made for general styling and then three modifiers for the width. Example is given in the 02_molecule folder

03_organism folder

Organism folder is for the block level elements. Elements that are being built from lego blocks or elements

04_Page folder

Page folder is for all the site-wide stylings that will be applied sitewide

Bower components folder

Bower components folder is the folder automatically created by the Bower package manager and it contains all the packages installed by the developer

Dist folder

Dist folder contains compiled css and custom js files. It also containes minified js and css files if we decide to minify it which is described Here

Bower.json file

Bower.json is the configuration file for the Bower which contains all the dev dependencies and with which you will control the packages we install by Bower and update them. Please review this section for more details

Gulpfile.js file

Gulpfile.js is the configuration file for the gulp task runner. Please visit this section for more details

package.json file

Package.json is the configuration and the beating heart of this kit. Please visit this section for more details

Important!

There is a certain prerequisite of using this kit when we download it or clone it via github into the sitecore project and that is we need to run a command which will install all our dependencies for us and set everything up for us. This needs to be run inside wherever the package.json is located. Package json looks like this . If we inspect the screenshot we will see that we have Dev dependecies. These are actually all the packages used currently. From minification to compilation to gulp itself. To install everything with one liner is to just run npm install where that package.json is located and you are ready to work.

If for any reason you get the error that you are missing bindings which usually happens because your environment has changed since running `npm install`. Run `npm rebuild node-sass` to build the binding for your current environment.
