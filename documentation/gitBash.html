Git Bash installation on Windows:

  • Download the Git bash installer from the Git website.

  • Run the installer from the exe file you have got.

  • Follow the prompts in the installer (Accept the license agreement, click the NEXT button a bunch of times and accept the default installation settings)

  • Restart your computer if prompted. Git bash is really important because it will give you a terminal with which you will be actually able to work

Test the gitbash by clicking on the icon after the installation or by hitting Windows key + S and searching for Git bash

When you open the gitbash you should see something like this.

Git on OSX

  • Git comes preinstalled on OSX, you can check the version by running git version in the terminal. If for any case it is not preinstalled or you haven't yourself installed it via package manager or directly from the Git website please contact me and we will do it together.

Important!

There is a certain prerequisite of using this kit when we download it or clone it via github into the sitecore project and that is we need to run a command which will install all our dependencies for us and set everything up for us. This needs to be run inside wherever the package.json is located. Package json looks like this . If we inspect the screenshot we will see that we have Dev dependecies. These are actually all the packages used currently. From minification to compilation to gulp itself. To install everything with one liner is to just run npm install where that package.json is located and you are ready to work.

If for any reason you get the error that you are missing bindings which usually happens because your environment has changed since running `npm install`. Run `npm rebuild node-sass` to build the binding for your current environment.
