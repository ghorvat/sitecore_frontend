Gulp task runner and installation

  • There are a lot of tasks involved with creating and deploying a front-end application. Common tasks include things like watching file changes, concatenating/minifying files, prefixing files for different browsers, and linting Javascript.

  • Installation of gulp is straightforward. It is simple as running npm install gulp -g. This will install it globally on your personal computer so you can use it from within terminal.

  • We will be using gulp for several things from compiling our sass code, watching for any changes and compiling to minifying js and css and optimising images. These are just but a few examples what gulp can do but it can do much much more then that.

Why should we use gulp?

While graphic tools like Koala are good enough for one time compiling I have an opinion that graphic tool will never be as powerfull as terminal which is actually the case. Currently we have several basic and most important things created for usage in the gulpfile.js. https://goo.gl/iU2WE2. This will be regulary maintained and updated by Kirk and Myself mainly but all other Senior or even Junior Frontend developers who know the ways of gulp are welcomed to add their input in the gulpfile.

  • Compile of sass and error handling gulp compile

  • Watch task that you can fire up while developing and it will automatically watch for any changes in any sass file and automatically compile the style.scss to style.css gulp watch

  • Minification of css gulp minify-css

  • Minification of js gulp compress

All of this tasks are ran through the terminal from the inside of the folder where the gulpfile.js is located considering you have the gulp already installed globally if you do not there is also a way we prepared for the gulp to install in just the project folder which i will explain right now

Important!

There is a certain prerequisite of using this kit when we download it or clone it via github into the sitecore project and that is we need to run a command which will install all our dependencies for us and set everything up for us. This needs to be run inside wherever the package.json is located. Package json looks like this . If we inspect the screenshot we will see that we have Dev dependecies. These are actually all the packages used currently. From minification to compilation to gulp itself. To install everything with one liner is to just run npm install where that package.json is located and you are ready to work.

If for any reason you get the error that you are missing bindings which usually happens because your environment has changed since running `npm install`. Run `npm rebuild node-sass` to build the binding for your current environment.
