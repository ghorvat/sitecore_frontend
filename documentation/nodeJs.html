NODE JS Installation Steps Windows:

  • Download the Windows installer from the Nodes.js website.

  • Run the installer (the .msi file you downloaded in the previous step.)

  • Follow the prompts in the installer (Accept the license agreement, click the NEXT button a bunch of times and accept the default installation settings).

  • Restart your computer. You won’t be able to run Node.js until you restart your computer.

Test Node JS

To see if Node is installed, open the Windows Command Prompt, Powershell or a similar command line tool, and type node -v. This should print a version number if everything has been installed correctly.

Test NPM

To see if NPM is installed, type npm -v in Terminal. This should print NPM’s version number. Optionally create a test file and run it. A simple way to test that node.js works is to create a JavaScript file: name it hello.js, and just add the code console.log('Node is installed!');. To run the code simply open your command line program, navigate to the folder where you saved the file and type node hello.js. This should output the content of the console log in the terminal proving that it is working.

NODE JS Installation Steps MAC/OSX:

Since OSX operating systems are already unix type systems they have a great terminal out of the box but due to the recent homebrew issues with NPM (Which i personally did not experience) we will install this manually same as on Windows. Please navigate to the Nodes.js website and download the apropriate dmg file and follow through the install. After install at the end you are prompted to make sure that /usr/local/bin is in your path, double check you have it by running in the Terminal: echo $PATH If not add it in either .bash_profile or .bashrc in your home directory.

Congratulations

If everything went well you should have NODE JS installed which is going to be really important for the rest of our frontend workflow. If you are experiencing any issues do not hesitate to ping Andri or Pieter :D

Important!

There is a certain prerequisite of using this kit when we download it or clone it via github into the sitecore project and that is we need to run a command which will install all our dependencies for us and set everything up for us. This needs to be run inside wherever the package.json is located. Package json looks like this . If we inspect the screenshot we will see that we have Dev dependecies. These are actually all the packages used currently. From minification to compilation to gulp itself. To install everything with one liner is to just run npm install where that package.json is located and you are ready to work.

If for any reason you get the error that you are missing bindings which usually happens because your environment has changed since running `npm install`. Run `npm rebuild node-sass` to build the binding for your current environment.
