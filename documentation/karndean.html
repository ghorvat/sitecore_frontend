Karndean Project

Karndean project stands out in a bad way from everything i have written before. While it has perfectly valid code the folder structure is in a really bad shape. This will be rectified soon enough by myself tomorrow i think. It still will not have the same structure as I have described before. The structure and the flow you have read beforehand is what we should aim for to improve our frontend.

Main starting point for Karndean and any work on it is Master branch. The flow is the following:

  • Branch out from master into a branch named by the task or feature you are working on for example: git checkout -b KASU-216

  • Continue working in the branch on your feature or a bug fix and when you are finished push it to the remote (github) with the following git add . This will add all the files and prepare them for commit. If you have a lot of stuff to add in the terminal and you are unsure what to add, add only the files you are sure off with the following git add path/to/file/something.css path/to/file/something.js path/to/file/something.css. You can add multiple files by appending them to the end and having a space between them.

  • After you have added everything please commit it with a semantical message of what you have done Git commit "Kasu-216 bug"

  • And in the end push it to remote with git push origin KASU-216

Now with github workflow explained lets get to the specifics

Current work for the Finder&Moodboard functionallities is being done in the finderAndMoodboard branch

To setup the current environment on the Karndean please do the following:

  • git clone git@github.com:LabDigital/Karndean.git (i suppose you have added your ssh keys to the github) This will clone the whole repository to your local environment

  • git checkout finderAndMoodboard(or any other branch)

  • Work on your fix

  • git add . or specific files as explained above

  • git commit "Explanation of the work done"

  • git pullJust to get all the changes other developers might have pushed. There are several scenarios that can occur while doing this

    • Git conflict - this is inevitable if multiple developers are working on the project and it will occur.

    • Noone pushed anything and the branch is up to date and you are good to push

    • Merge request occurs when someone did additional work but git can merge everything without conflicts. You will ussually get a question in the terminal if you want to merge your changes please use the vi or vim or GUI write and quit commands respectively depending on which you are using.

  • And finally do a git push. There is no need to specify the branch but just in case if git refuses to push to remote write git push origin finderAndMoodboard or any name of the branch you are working on

  • Do not forget to pull always before you push if more developers are working ona fix except you.
